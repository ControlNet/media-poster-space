# Learnings

- 2026-03-13 T1: Shared-core wall eligibility can stay fully deterministic with synthetic rect fixtures; strict overlap checks (`right > leftEdge`, `left < rightEdge`, `bottom > topEdge`, `top < bottomEdge`) make the first visible pixel protected and boundary equality count as full exit.
- 2026-03-13 T1: Entry-side eligibility is direction-specific (`tile.left >= viewport.right` for normal rows, `tile.right <= viewport.left` for reverse rows), and the final writable buffer should be distance-sorted back toward the viewport and capped to `WALL_POSTER_ENTRY_BUFFER_LIMIT` (2).
- 2026-03-13 T2: The backlog contract stays deterministic when pending items are rebuilt as `dedupe(pending) + new incoming not present in previous/current identities`; that preserves FIFO order while avoiding duplicate media across prior stream items, next stream items, and already-queued backlog entries.
- 2026-03-13 T2: The shared-core stream contract can expose `applied` / `deferred` / `noop` / `unavailable` as an explicit result while keeping the DOM key wrapper boolean-compatible, so host runtimes keep their existing truthiness fallback behavior until they adopt the richer contract.
- 2026-03-13 T2: Deferred poster streaming needs a FIFO `pendingIncomingItems` queue that dedupes existing backlog identities first, then appends only truly new next-stream identities; otherwise posters disappear when ingestion advances past an unapplied item.
- 2026-03-13 T3: The shared-core applier can safely preserve visible poster identities by taking a fresh per-row geometry snapshot on each apply pass, deriving `orderedWritableIndices` from live rects, and consuming per-pass writable orders separately from row round-robin row selection.
- 2026-03-13 T3: The shared-core applier can stay DOM-light by snapshotting each tile from the current `tile` object via `getBoundingClientRect()` when present and otherwise accepting a structural rect-shaped object; Vitest can then deterministically stub `window.innerWidth/innerHeight` plus synthetic tile rects without jsdom.
- 2026-03-13 T3: Bounded entry-buffer writes work cleanly when each apply pass computes per-row `orderedWritableIndices`, tracks consumed writable orders for that pass, and reuses `nextEntryPointer` only as a cursor within the current writable snapshot; this keeps visible/intersecting tiles frozen while preserving cross-row round-robin distribution and FIFO backlog draining.
- 2026-03-13 T4: Resize/fullscreen safety stays deterministic when each apply pass resolves row eligibility from the current viewport + tile geometry snapshot rather than any cached writable-slot list; viewport growth can then shrink future writable buffers without touching tiles that became visible.
- 2026-03-13 T4: Viewport contraction and reverse-row motion both reclassify entry slots correctly when tests mutate only synthetic rect snapshots and `window.innerWidth`, proving that fully exited tiles become writable again while newly intersecting left-entry tiles remain protected.
- 2026-03-13 T5: Queue/ingestion integration can prove healthy deferred writes without host remount logic by pairing the existing onboarding controller harness with shared-core `applyWallPosterGridStreamState`, synthetic row geometry, and an underflow refill payload; `ingestionStatus` stays `ready` and no `queue.refill-failed` event is emitted while pending posters remain queued.
- 2026-03-13 T5: Deferred apply passes still advance `nextIncomingRowPointer`, so FIFO drain assertions must account for the current row pointer when entry slots reopen; the pending backlog order stays intact, but the first reopened slot may belong to a later row in the round-robin cycle.
- 2026-03-15 T7: Desktop host parity needs an actual mounted-poster readiness signal (`wall-poster-grid` buttons), not `poster-item-*` sentinels; healthy shared-core `deferred`/`noop` results should stay mounted once poster tiles exist, while the initial empty-grid defer still needs a one-time fallback remount so the populated wall can mount and later broken-readiness incidents can rearm after recovery.
- 2026-03-15 T6: The web host can treat shared-core `applied` / `deferred` / `noop` results as healthy in-place outcomes so long as fallback readiness checks look for real mounted poster tile buttons inside `[data-testid="wall-poster-grid"]`, not the `poster-item-*` probe sentinel family; that preserves empty-grid-to-populated remounts without false remounts when a sentinel disappears.
- 2026-03-15 T6/T7 verification: Web host acceptance required sequential Playwright execution (`--workers=1`) to avoid cross-run interference on the shared Vite test server; once run cleanly, the healthy-stream remount, clock, and fullscreen probes all passed without additional code changes.
- 2026-03-15 T6/T7 verification: Desktop host parity is covered by both targeted `onboarding-auth` tests and the full desktop auth suite; the healthy-mounted and broken-readiness rearm cases pass alongside the existing broader onboarding/runtime regressions.
- 2026-03-15 T8: Web Playwright coverage can prove visible-poster immutability without `poster-item-*` sentinels by assigning stable probe ids to real `button` tiles inside `[data-testid="wall-poster-grid"]`, sorting currently visible tiles by viewport intersection area, and comparing each tile's poster-thumb `backgroundImage` before/after a healthy manual refresh.
- 2026-03-15 T8: To show the refresh actually streamed new posters, the spec can pair the visible-tile background freeze assertion with a second check that at least one non-tracked grid button changes `backgroundImage`; this keeps the test focused on real visible identities while allowing off-screen entry tiles to update.
- 2026-03-15 T9: Desktop jsdom parity can mirror the web visible-poster freeze proof by stubbing `getBoundingClientRect()` on selected real `button` tiles inside `[data-testid="wall-poster-grid"]`, assigning stable probe ids on those mounted buttons, and comparing each tile's poster-thumb `backgroundImage` before/after a healthy manual refresh; unstubbed jsdom buttons stay non-visible while explicitly off-screen tiles remain writable entry slots that prove the refresh changed only non-visible posters.
- 2026-03-15 T10 blocker: Web idle-hide should be re-armed by closing diagnostics itself, not by a follow-up pointermove; scheduling idle hide on the diagnostics open->closed transition fixes the contract while keeping diagnostics-open suppression intact and lets the Playwright probe assert hidden controls without synthetic post-close interaction.
- 2026-03-15 T10 flake cleanup: the web healthy-stream/remount Playwright spec was racing the manual refresh control while a prior refresh was still rendering; polling the real button DOM state (visible + not disabled) inside the shared web E2E helper removed the retry-only failure without touching runtime behavior.
- 2026-03-15 T10 onboarding-finish flake: in the healthy-stream/remount web spec, waiting for the post-login library checkbox to be visible and checked before clicking onboarding-finish removes the first-attempt race under the full 8-worker e2e gate without changing runtime behavior.
- 2026-03-15 T10 remembered-login flake: the web remembered server/username restart spec also needed the hardened post-login library readiness contract before reload; waiting for library-checkbox-movies-main to be visible and checked with an explicit timeout removed the retry-only race in both targeted and full-file Playwright runs without touching runtime code.
- 2026-03-15 T10 remembered-login refinement: the most reliable fix for the remembered server/username restart spec is to wait for the preflight GET /System/Info/Public response before filling credentials, then still gate on library-checkbox-movies-main being visible and checked before reload; the earlier first-attempt failure was an onboarding-form rerender race, not just a slow library checkbox.
- 2026-03-15 F4 scope lesson: when a reviewer rejects a behavior fix as out-of-scope, prefer restoring the original runtime contract and moving coverage back into the affected host test flow rather than broadening shared callback semantics; the web diagnostics-idle spec can prove preserved behavior by re-arming idle-hide with a post-close pointer interaction instead of making diagnostics close itself schedule idle hide.
